✨ FUNCIONALIDADES

═══════════════════════════════════════════════════════════════════════════════════

✅ Multi-Critério Independente
   Cada equipamento pode ter até 3 critérios de manutenção funcionando
   simultaneamente. Não é necessário usar todos.

✅ Status Inteligente
   O sistema calcula qual critério é mais URGENTE e mostra como
   "Status Geral". Exemplo: se horímetro venceu mas km tá ok,
   exibe 🔴 VENCIDA (via horímetro).

✅ Cores Indicam Urgência
   🔴 VENCIDA — vermelho (> 100h/km atrasada)
   ⚠️ PRÓXIMA — amarelo (≤20% do intervalo para vencer)
   ✅ EM DIA — verde (confortável)

✅ Formulário Dinâmico
   Ao selecionar equipamento, popula automaticamente valores salvos
   anteriormente (horimetro, hodometro, períodos)

✅ Validação Segura
   Exige pelo menos 1 critério preenchido para salvar
   Supabase REST via browser seguro (autenticação por chave anônima)

📞 TROUBLESHOOTING

═══════════════════════════════════════════════════════════════════════════════════

P: "Campos não salvam no Supabase"
R: Migração SQL não foi executada. Siga passo 1️⃣ acima.

P: "Formulário mostra erro ao salvar"
R: Abra F12 > Console > procure por erro de fetch HTTP
   Verifique se supabase_url e supabase_anon estão configurados

P: "Tabela está vazia"
R: a) Verifique se há equipamentos em MANUTENCAO (dados brutos)
   b) Abra Supabase > manutencoes_equipamentos > verifique registros
   c) Cheque permissões RLS (Row-Level Security)

P: "Status não atualiza quando seleciono equipamento"
R: Recarregue página (F5) para carregar dados novos do backend
